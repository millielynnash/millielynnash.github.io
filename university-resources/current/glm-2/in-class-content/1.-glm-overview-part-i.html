Review of Descriptive and Inferential Statistics

  • Descriptive Statistics
    • Descriptive statistics 'describe' the state of the data.
    • More clear-cut, though with data transformation this can become complex.
  • Inferential Statistics
    • Inferential statistics 'infer' information about the participants based on their data.
    • Can be descriptive but also predictive.
  • 'Frequentist' Inferences
    • Statistical inferences based on frequencies of probabilities.
    • Basis for Null Hypothesis statistical testing (NHST), statistical significance, Confidence Intervals, etc.

Levels of Measurement

  • Nominal Variable or Nominal Variable
    • Categories that are distinct and which do not have numerical value (e.g., school)
  • Ordinal Variables
    • Numerical values that only signify ranking (e.g., risk level)
  • Interval Variable
    • Numerical data that has equivalent ranges between values but no meaningful zero (e.g., SAT scores, percentile rankings, etc.)
  • Ratio Variable
    • Same as interval, but with a meaningful zero (e.g., final exam scores)

General Linear Model (GLM)

  • Linear = additive
  • General model = linear in the parameters
  • Expected that the variation of variables will be 'normally distributed' (Normal Distribution)
  • Beta's (b's) are regression weights that are optimally estimated based on their contribution of each variable to reasonable predictions of Y.
  • Absolute value of b depends on the metric scale of the x variable that they modify.

Generalized Linear Model (GLzM)

  • Extension of the GLM
  • The outcome variable follows different distributions
    • I.e., probability distributions, binomial distributions, gamma distribution, poisson distributions, etc.
  • The outcome variable is non-linear (not continuous)
  • Binary/dichotomous, count data, etc.
  • The outcome variable is traditionally related to the linear model through link functions (such as transformations)

Analysis Selection Considerations

Design Considerations

  • What are your research questions/hypotheses?
  • Is your design correlational, experimental, or quasi-experimental?
  • Is your study cross-sectional or longitudinal?
  • Are assumptions of analyses being met a priori?
  • Regression Analysis
    • Linearly related and normally distributed
  • Analysis of Variance (ANOVA)
    • Independent observations, normally distributed, homogeneity (equal) variance across groups

Data Considerations

  • Familiarity with the dataset of your choosing
  • How many cases?
  • How many variables?
  • Levels of measurement?
  • Who are the participants?
  • Is data identifiable?
    • If not, is there a classification system in place?
  • Any special considerations?

SPSS Introduction

  • Two methods of navigating SPSS
    • Point and click
    • Syntax
  • A combination of both methods can produce a highly efficient (and less stressful) experience when using the software.
  • Point and click menus are intuitive and clear for new users
  • Syntax allows for a record of what has been done that can be easily reproduced and communicated with peers.

SPSS Windows

  • Data editor
    • See your data and variables directly
    • Can carry out statistical functions
  • Output viewer
    • See the results of your analyses
  • Syntax editor
    • Enter text commands directly or save them from the data editor
    • Analyze -> Descriptives -> Paste

SPSS Syntax

  • Formatting
    • All statements end in a period (red text = bad; blue text = good)
    • Comment to yourself by starting a line with * and end with a period .
    • Not case-sensitive
    • You can highlight any or all of a command to run it
  • Structure
    • Use the numeric lines to keep track of things
    • Order the syntax logically and be sure to categorize!

Generating Basic Syntax

  • Using the Point and click method to create the desired syntax
    • Descriptive statistics of variable(s)
      • Analyze -> Descriptives -> Descriptives
  • Using syntax
    • File -> New -> Syntax
    • Click on Window -> Designated Syntax Window
    • Example
      • Compute variable from existing variable
        • Subtracting 1 from the value of the variables such as 'Whether mom graduated HS' with the original values being 1 'did not graduate' and 2 'graduated HS' to create dummy variable.
        • COMPUTE hs_adj = mom_hs - 1
        • COMPUTE age_squared = mom_age**2
          • To square a value in syntax, include a double star (**) and 2
        • EXECUTE
        • Highlight the computations and click RUN
    • Commenting on Syntax
      • Type the asterisks sign * computing new variable from an existing variable
      • Even if the comments are highlighted, they will not be executed since they are just comments.
    • To recode a variable (You can also do it manually from the variable view)
      • RECODE mom_hs (2=1)(1=0) into new_mom_hs
      • VARIABLE LABELS new_mom_hs "new mother's high school status"
      • EXECUTE
    • To create descriptive statistics
      • DESCRIPTIVES VARIABLES mom_age kid_score
      • /STATISTICS = MEAN STDDEV MIN MAX
      • Highlight the computations and click RUN
    • To create new group variable from existing variable (replacing variables with dummy coding)
      • IF mom_age > 20 Mom_over = 1
      • IF mom_age <= 20 Mom_over = 0

Brief Introduction to R-Studio

  • Upper left quadrant is the 'Source'
  • Upper right quadrant is the 'Environment/History'
  • Lower left quadrant is the 'Console'
  • Lower right quadrant are the 'Files/Plots/Packages/Help/etc.'

Source

  • This just looks like a notebook where you write your code
  • You can save this and share the file with your project mates
  • You can make comments by beginning a line of code with '#'
  • You can call it your R Script/Source
  • You can create a new script by clicking the green '+' symbol on the top left corner and selecting 'R Script'
  • To run a particular line of code, click 'run' or use 'Command + return' for Macbook and 'Control + Enter' for Windows

Console

  • This is your 'Console'
  • This is where the code you run from the source gets evaluated
  • You can directly type your code here and get it evaluated, but the code will not be on your source/R Script

Environment / History

  • Shows all objects in your working space (e.g., your data, matrices, vector, functions, etc.)
  • You can import data directly by using the 'Import Dataset'
  • You can use the 'Broom' to clear all objects from the workspace.

Files / Plots / Packages / Help

  • You see your file directories
  • You can view your plots
  • You can see your 'Packages' and also install new packages
  • You can read more about the function under 'Help'

Important to Do before Beginning to Write Any Code or Do Anything

  1. Make sure your Source file (R Script) and your dataset are saved (or located) in the same folder on your computer
  2. Set your working directory
    • Steps:
      • Click 'Session' -> Choose 'Set Working Directory' -> Select 'To Source File Location'

Importing a Dataset Using the 'Point-and-Click'

  • First approach
    • Click on 'Files' -> Select 'Import Dataset' -> Choose the data you want to import (e.g., Excel, SPSS data, STATA data, etc.)
  • Second approach
    • Click on 'Import Dataset' under 'ENVIRONMENT' -> Choose the data you want to import (e.g., Excel, SPSS data, STATA data, etc.)

Base R and Installing Packages

  • R comes with some functions which are free to use (e.g., mean (), sd ())
  • You do not need to install any package before using those functions
  • However, to access a function written by others (other than the original 'authors' of R), you first have to install it as a 'package.'
  • These packages often come with dataset(s), functions, examples, etc.
  • Install a package using 'install.Packages ('name of the package)')'
  • You need to install the package once and it will be on your computer permanently.

Loading a Package after Installing It

  • After installing a package, you need to load it before having access to its functions, datasets, etc.
  • Use 'library(name of package)' to load a package.

Exploring Your Data

  • There are many functions to explore your data
    • View to have a quick view of your data
    • Head to print out the first 6 rows
    • Tail to print out the last 6 rows
    • Summary to look at the summary of all your variables
    • Psych::describe to check the descriptive statistics
    • Use the dollar sign ($) to help select a specific variable
      • E.g., mean(kids_score $age) will compute the mean for the variable age in the 'kids_score' dataset
    • mean() to find the mean of a variable
    • sd() to find the standard deviation
    • var() to find variance
    • sum() to find the totals
    • Use '?' to learn more about a function
    • nrow(data) to find the number of rows
    • ncol(data) to find the number of columns
    • colnames(data) to print the names of the columns (variables)
    • str(data) to look at the structure of the variables
    • table() to print out the frequency in each group for a Nominal Variable
    • min() and max() to find the minimum and maximum of a variable
