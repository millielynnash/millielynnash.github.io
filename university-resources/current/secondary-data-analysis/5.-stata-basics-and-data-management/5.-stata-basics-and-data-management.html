5. STATA Basics and Data Management

Overview of STATA

Why Use STATA?

  • Great for data management
  • Very powerful and flexible, for most models.
  • STATA can
    • Handle Continuous, Categorical, and count variables
    • Evaluate group differences via multiple group analysis
    • Evaluate complex survey data and make necessary adjustments (e.g., clustering, weights)
    • Handle missing data through full information maximum likelihood or multiple imputations
    • Run more basic (Regression, EFA, CFA) and more complex analyses (MLM, LGCM, GMM)

Navigating Around STATA

Results Window

  • The big window
  • Results of all STATA commands appear here (except graphs)

Command Window

  • Records all STATA commands that have been entered. A previous command can be repeated by double-clicking the command in the Review window

Variables Window

  • Shows a record of all variables in the dataset

Toolbar

  • Across the top of the screen

Spreadsheet

  • Click the (editor) button. All data are visible here.
  • Note that no commands can be executed when the data editor is open.

More Navigation

  • There are two ways to operate STATA

Interactive Mode

  • Commands can be types directly into the Command window and executed by pressing Enter.

Batch Mode

  • Commands can be written in a separate file (called a do-file) and executed together in one step.

Do Files

  • List of all the commands you need for your project
  • No need to repeat work; recreate your dataset by running the do-file
  • Easy to correct mistakes you find later
  • To create / edit do-file:
    • Starting a new do-file: doedit
    • Editing an existing do-file: doedit DOFILENAME.do
    • Type in commands
    • Save do-file (control + s)
  • To use
    • Do DOFILENAME.Do
  • Try to be organized
    • Use headings
    • For all test in do-flies that are not commands, begin the line with *

Log Files

  • Log records all commands and output during the session
  • To use:
    • Open new log
      • Log using LOGNAME.log / cmdlog using CLOGNAME.log
    • Write over an existing log
      • Log using LOGNAME.log / cmdlog using CMDLOGNAME.log, replace
    • Add to an existing log
      • Log using LOGNAME.log / cmdlog using CMDLOGNAME.log, append
    • To close log
      • Log close / cmdlog close

Reading and Saving Data

  • To read a file using the command window
    • Use "C:...\sample.dta", clear use sample.dta, clear
  • To save a file
    • Save "C:...\sample.dta" [, replace]

Examining Data

Browse Command

  • Type 'browse' in the command window
  • Column holds the variable
  • Row holds the observation
  • Cell of a particular variable for a particular observation is a value

Edit Command

  • Allows you to edit the data

Describe Command

  • The number of observations in the dataset
  • Number of variables in the dataset
  • The amount of memory the dataset is using and how much memory you still have to work with
  • Basic information about the variables in our dataset

Codebook Command

  • Variable name, label and type
  • Some basic descriptive statistics for variable

Understanding Command Syntax

  • The general syntax for all STATA commands is:
    [prefix:] cmdname [varlist] [=exp] [if exp] [in exp] [weight] [using filename] [, options]
  • Elements in square brackets are optional for some commands
  • Sometimes cmdname is all that is required, for example, codebook or describe
  • STATA is case sensitive

Variable Transformation

  • New variables can be created using generate
    • Gen racev2 = race
  • New variable can contain functions or combinations of existing variables
    • Gen income = inc1 + inc2 + inc3
    • You can use *, +, -, /, ^
  • Replace may be used to change the contents of an existing variable
    • Replace oldvar = exp1 [if exp2]
  • Always recommend that you do not save over original values.
  • 'If' is used to restrict the command to a desired subset of observations:
    • Replace race =. If race == 999
  • The double == sign is used to test for equality, while the single sign is used for assignment
  • Operators can be used after if:
    • & denotes 'and'
    • | denotes 'or'
    • ~ or ! Denote 'not'
  • Rename may be used to rename variables
    • Rename oldvarname newvarname
  • To drop a variable or variables, type
    • Drop varlist
  • Alternatively, keep varlist eliminates everything but varlist
  • To drop certain observations, use
    • Drop if exp
    • For example, drop if unemplrate==

Combining Datasets

  • Dataset in memory is called 'master dataset'
  • Dataset filename is called 'using dataset'
  • Variables with same name will be combined
  • Variables in only one dataset will have missing values for observations from the other dataset.
  • To join corresponding observations from a STATA dataset with those in the dataset in memory, type:
    • Merge 1:1 varlist using filename
  • If more than one observations has the same value of varlist in the master dataset, use:
    • Merge m: 1 varlist using filename
  • To add another STATA dataset below the end of the dataset in memory, type:
    • Append using filename

Descriptive Statistics

  • For Mean, Standard Deviation (SD) use sum:
    • Sum bex_tm54
  • For stats by subgroup, add bysort: in option section
    • Bysort csex_m01: sum bex_tm54
  • For Categorical Variable s use tab:
    • Tab csex_m01

Correlation Syntax

  • Syntax using correlate
    • Correlate bex_tm54 bex_tm1s bex_tmg3
    • pwcorr provides significance levels

T-Test Syntax

  • Syntax using ttest
    • ttest bex_tm54, by (csex_m01)

Linear Regression Syntax

  • Results using regress:
    • Regress bex_tm 54 meducm 01 inc_6_54 csex_m01
      • Coef. = Beta Coefficient
      • Std. Err. = Standard error
      • T = Ratio of beta to standard error
      • P>|t| = significance level
      • [95% conf. Interval] = Confidence Interval
    • Critial Ratio
      • The most basic form of a statistical test of significance
      • 1.96 for p < .05
      • 2.58 for p < .01
      • 3.24 for p < .001

Where to Get Help

  • If you know the name of a command, enter
    • Help cmdname
  • If you don't know it, enter
    • Findit word 1 [word2] ...
    • This queries a database and some of the official internet sources (such as STATA FAQs, STATA articles)
  • Google
  • Email or call STATA Technical services
  • Statalist archives / forums
  • Check the cheat sheets (statacheatsheets.pdf)
